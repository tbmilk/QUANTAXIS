# QUANTAXIS 实盘第一阶段规划与任务书

更新时间: 2026-04-18
范围: 第一阶段只覆盖 `期货(CTP)` 与 `A股(QMT)`
原则: 先跑通期货实盘闭环，再最大化复用通用模块实现 A 股 QMT 实盘

## 已确认决策

1. CTP 接入方案固定采用 `openctp`
2. 通用抽象必须前置，不允许先写死期货实现再回头大改接口
3. 现有 MongoDB 历史数据作为第一阶段标准测试行情源和回放源
4. Rust 主链路固定为 `openctp + ctp2rs(upstream)`，以减少自研 FFI 工作量
5. 第一阶段 CTP 验收环境优先采用 `openctp TTS`，不再以 `SimNow` 作为主验收环境
6. Python `ctpbee` 桥接仅保留为降级/应急预案，不作为第一阶段主交付路线

---

## 1. 目标重述

第一阶段不追求“多市场全自动盈利”，只追求两件事:

1. 跑通 `期货 CTP` 实盘底座
2. 在通用模块复用的前提下，补齐 `A股 QMT` 实盘

第一阶段完成后，系统应具备:

- 实时行情接入
- 信号生成
- 风控拦截
- 委托下发
- 回报处理
- 持仓/账户同步
- 数据持久化
- 对账与监控

不在第一阶段范围内:

- 港股实盘
- 期权实盘
- 跨市场套利自动化
- 期权定价与 Greeks
- 多腿原子套利执行

- `A股 QMT` 放在第二优先级，但应尽量复用期货阶段沉淀出来的通用模块
- 第一阶段不要把“策略 alpha”与“执行底座”混在一起，先做底座

## 3. 架构原则

第一阶段固定采用下面的分层:

### 3.1 Python 职责

- 历史数据抓取与入库
- 回测与研究
- 策略原型验证
- QMT 桥接辅助进程

### 3.2 Rust 职责

- 实时行情接入
- 行情分发
- 指标计算
- 信号路由
- 风控评估
- OMS
- 柜台适配
- 状态持久化

### 3.3 通用抽象

第一阶段必须优先抽象这些通用接口，再做市场接入:

- `MarketDataSource`
- `BrokerAdapter`
- `OrderRouter`
- `OmsStore`
- `LiveContext`
- `RiskService`
- `SignalGenerator`

这样做的原因是:

- CTP 与 QMT 只在“行情入口”和“下单出口”不同
- 中间链路应该完全复用

### 3.4 测试数据原则

第一阶段统一采用三类数据源:

1. `MongoDB 历史数据`
   - 用于回放、联调、接口验证、回归测试
2. `CTP/openctp 模拟盘实时行情`
   - 用于实时链路验证
3. `QMT 模拟环境`
   - 用于 A 股交易链路验证

其中 `MongoDB 历史数据` 是第一优先测试源，原因是:

- 仓库里已经有大量历史数据
- 数据格式与 Python 侧现有回测/研究链路一致
- 可用于构造稳定、可重复的测试与回归环境

### 3.5 openctp 约束

第一阶段中，所有 CTP 相关任务统一以 `openctp` 为默认前提:

- 行情接入基于 `openctp` 行情接口
- 交易接入基于 `openctp` 交易接口
- 模拟盘联调用 `openctp TTS` 优先
- 如需兼容其他 CTP 发行版，放到第一阶段之后处理

### 3.6 CTP 依赖约束

第一阶段 CTP Rust 实现统一采用:

- 上游 `pseudocodes/ctp2rs`

不作为主依赖的方案:

- `yutiansut/ctp2rs` fork
- `WhisperCapital/openctp-rs`
- `yutiansut/openctp-rs`

原因:

- 继续使用 `ctp2rs` 可以显著减少第一阶段 CTP 落地成本
- 上游 `pseudocodes/ctp2rs` 已有 openctp/TTS 示例，可直接参考
- 自研 FFI 在当前阶段收益低于维护成本

### 3.7 降级策略原则

第一阶段保留一条 CTP 降级思路:

- Python `ctpbee bridge`

但它的定位仅为:

- 登录/环境联调的应急替代
- Rust FFI 主链路异常时的备用预案

不作为第一阶段必须并行交付的主功能，避免:

- 执行面过宽
- 测试矩阵翻倍
- 主链路长期悬空

---

## 4. 实施顺序

固定顺序如下:

1. 先完成通用抽象层
2. 再完成期货 `CTP(openctp)` 闭环
3. 最后接入 A 股 `QMT`

不允许:

- 先写死期货实现再回头抽象接口
- 先做 QMT 再补 CTP

原因:

- 仓库现有设计更偏期货/QIFI/CTP 语义
- 通用抽象不前置，后面接 QMT 时会导致大面积返工

## 8. 推荐的下一步

1. 先做 `P0-T01 固化统一接口`
2. 然后做 `P0-T03 建立 MongoDB 历史数据测试基线`

- MongoDB 历史数据可以尽早提供稳定回归环境
